# 项目文档规范

在项目开发过程中，涉及以下内容时应自动生成相应文档并存放在 `docs` 目录下。

## 需生成文档的场景

| 场景 | 文档类型 | 存放位置 |
|------|----------|----------|
| 新增功能模块 | 功能说明文档 | `docs/features/功能名.md` |
| 新增 API 接口 | API 接口文档 | `docs/api/接口名.md` |
| 新增数据结构 | 数据结构说明 | `docs/data/数据结构名.md` |
| 修改配置文件 | 配置说明文档 | `docs/config/配置名.md` |
| 业务流程变更 | 流程说明文档 | `docs/process/流程名.md` |

## 文档模板

### 功能说明文档
```markdown
# 功能名称

## 功能描述
简要描述这个功能是什么、解决什么问题。

## 使用方法
说明如何使用这个功能。

## 参数说明
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| param1 | string | 是 | 参数1说明 |

## 返回值
说明返回值的结构和含义。

## 示例
```python
# 示例代码
```
```

### API 接口文档
```markdown
# API: 接口名称

## 接口描述
简述接口的功能和用途。

## 请求地址
`POST /api/xxx`

## 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| param1 | string | 是 | 参数说明 |

## 响应示例
```json
{
  "code": 0,
  "data": {}
}
```

## 错误码
| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
```

## 目录结构
生成的文档应按以下目录结构组织：
```
docs/
├── api/           # API 接口文档
├── config/        # 配置说明文档
├── data/          # 数据结构文档
├── features/      # 功能说明文档
├── process/       # 流程说明文档
└── README.md      # 文档索引
```
